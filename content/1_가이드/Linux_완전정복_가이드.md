## Linux 개요 및 역사

### 📚 Linux 탄생과 발전

#### 탄생 배경
- **1991년 9월 17일**: Linus Torvalds가 첫 Linux 커널 릴리즈
- **개발 동기**: 핀란드 헬싱키 대학교 학생이었던 리누스가 MINIX(Unix 기반)를 사용하던 중, **무료이면서 수정 가능한** 운영체제 커널을 만들기 위한 **취미 프로젝트**로 시작

#### 발전 단계

**Pre-1991: Unix의 기초**
- **1970년대**: AT&T Bell Labs에서 Unix 운영체제 개발
- Unix는 엔터프라이즈 환경의 기반이 됨

**1991: Linux의 시작**
- **1991년 8월 25일**: Linus Torvalds가 Usenet 뉴스그룹에 Linux 커널 첫 공지
- **무료 운영체제 커널 개발**이라는 취미 프로젝트로 시작

**Mid-1990s: GNU GPL의 영향**
- **GNU General Public License (GPL)** 채택
- **자유로운 배포와 소스코드 수정 허용**
- 개발자 및 사용자 커뮤니티 형성으로 다양한 기능 추가와 배포판 생성

**Late 1990s ~ 현재: 글로벌 확산**
- **취미 프로젝트 → 강력하고 다재다능한 운영체제**로 성장
- 개인용 컴퓨터부터 서버, 모바일 기기(Android), 임베디드 시스템, 슈퍼컴퓨터까지 광범위한 활용

### 🔄 Linux vs Unix 비교

| 구분 | Unix | Linux |
|------|------|-------|
| **개발** | AT&T Bell Labs | 커뮤니티 주도 |
| **라이선스** | 상업적 라이선스 (IBM AIX, HP-UX, Sun Solaris) | GNU General Public License (GPL) |
| **배포** | 제한적, 유료 | **자유롭게 사용, 수정, 배포 가능** |
| **용도** | 엔터프라이즈 환경, 견고한 안정성과 신뢰성 필요 | 슈퍼컴퓨터부터 라우터, 스마트워치 등 **광범위한 하드웨어** |

**핵심 특징**
- **Unix**: 주로 **엔터프라이즈 환경**에서 견고한 안정성과 신뢰성이 필요한 서버와 컴퓨터에 사용
- **Linux**: **다양한 하드웨어**에서 실행 가능한 범용성

## Linux 배포판과 시스템 구성

### 📦 Linux 배포판 (Distribution)

**배포판이란?**
- **Linux 커널 + 소프트웨어 컬렉션 + 패키지 관리 시스템**으로 구성된 완전한 운영체제
- 사용자는 Linux 배포판 중 하나를 다운로드하여 운영체제 설치

#### 🔴 Red Hat 계열
**대표 배포판**: Fedora, CentOS, RHEL (Red Hat Enterprise Linux)

**패키지 관리**:
- **RPM Package Manager (RPM)**: 소프트웨어 설치, 업데이트, 제거
- **상위 도구**: 
  - `yum` (기존)
  - `dnf` (최신) - 의존성 처리 및 소프트웨어 관리 간소화

**특징**: **상업적 시장** 중심

#### 🟡 Debian 계열  
**대표 배포판**: Ubuntu, Debian, Linux Mint

**패키지 관리**:
- **Debian 패키지 관리 시스템**: `.deb` 패키지
- **Advanced Package Tool (APT)**: 
  - 의존성 자동 처리
  - 손쉬운 시스템 업데이트 및 소프트웨어 관리

**특징**: **범용적** - 데스크톱과 서버 시장 모두 대응

#### 🔵 기타 주요 배포판
- **Arch Linux**: 고급 사용자용, 최신 소프트웨어
- **AlmaLinux**: CentOS 후속
- **Rocky Linux**: CentOS 후속

### 🏗️ Linux 시스템 구성요소

#### 1. 커널 (Kernel)
- **시스템의 핵심**
- 하드웨어 관리, 프로세스 실행, 시스템 콜 처리
- 운영체제의 가장 기본적인 부분

#### 2. 셸 (Shell)
- **사용자와 운영체제 간의 인터페이스**
- **CLI (Command Line Interface)**: 명령어 기반 텍스트 인터페이스
- **GUI (Graphical User Interface)**: 그래픽 기반 사용자 인터페이스

#### 3. 파일시스템 (Filesystem)
- **파일 조직 및 저장** 담당
- **지원 파일시스템**: ext4, XFS, Btrfs 등 다양한 파일시스템 지원

#### 4. GNU 도구 (GNU Tools)
- **시스템 관리, 프로그래밍, 사용자 작업**을 위한 소프트웨어 도구 모음
- Linux 커널을 보완하여 **완전한 운영체제** 구성

## 파일시스템 구조 (FHS)

Linux 디렉토리 구조는 **FHS(Filesystem Hierarchy Standard)**를 따르는 계층적 파일시스템 레이아웃으로, 각 디렉토리가 특정 목적을 가지고 있습니다.

### 🚀 시스템 부팅 관련

#### `/boot` - 시스템 부팅 파일
- **용도**: 시스템 부팅에 필요한 모든 파일
- **포함 내용**:
  - Linux 커널
  - 초기 RAM 디스크 이미지 (드라이버용)
  - 부트로더 설정 (GRUB 등)

### 🔧 시스템 설정 및 관리

#### `/etc` - 시스템 설정 파일
- **용도**: 시스템 전체 설정 파일들
- **특징**: 텍스트 파일로 편집 가능
- **포함 내용**:
  - 호스트명 설정
  - 사용자 권한 설정 (sudoers)
  - 애플리케이션별 설정

#### `/dev` - 디바이스 파일
- **용도**: 하드웨어 장치에 대한 인터페이스
- **특징**: Linux는 장치를 파일로 취급
- **예시**: 하드드라이브, USB 장치 등

### 💻 실행 파일 (바이너리)

#### `/bin` - 기본 사용자 바이너리
- **용도**: 부팅 및 시스템 복구에 필요한 필수 명령어
- **사용자**: 모든 사용자
- **예시**: `ls`, `cp`, `mv` 등

#### `/sbin` - 시스템 바이너리
- **용도**: 시스템 관리용 필수 바이너리
- **사용자**: 주로 root 사용자
- **예시**: `fdisk`, `ifconfig`, `init` 등

#### `/usr/bin` - 사용자 바이너리
- **용도**: 부팅에 불필요한 사용자 유틸리티
- **예시**: 대부분의 사용자 애플리케이션

#### `/usr/sbin` - 사용자 시스템 바이너리
- **용도**: 부팅에 불필요한 시스템 관리 도구

### 📚 라이브러리

#### `/lib` - 필수 공유 라이브러리
- **용도**: `/bin`과 `/sbin`의 바이너리에 필요한 라이브러리
- **포함**: 커널 모듈

#### `/usr/lib` - 사용자 라이브러리
- **용도**: `/usr/bin`과 `/usr/sbin`의 바이너리용 라이브러리

### 💾 데이터 저장소

#### `/home` - 사용자 홈 디렉토리
- **용도**: 각 사용자의 개인 파일 및 설정
- **구조**: `/home/사용자명/`
- **포함**: 개인 파일, 사용자별 설정

#### `/var` - 가변 데이터
- **용도**: 크기가 변하는 파일들
- **포함 내용**:
  - 로그 파일 (`/var/log`)
  - 스풀 파일
  - 캐시된 데이터
- **중요성**: 모니터링 및 디버깅에 필수

#### `/tmp` - 임시 디렉토리
- **용도**: 애플리케이션의 임시 파일 저장
- **특징**: 재부팅 시 정리될 수 있음
- **관리**: 프로그램이 자체적으로 정리해야 함

### 🔄 런타임 및 가상 파일시스템

#### `/run` - 런타임 데이터
- **용도**: 부팅 초기부터의 일시적 상태 파일
- **포함**: 프로세스 ID, 락 파일 등
- **특징**: 재부팅 시 초기화됨

#### `/proc` - 프로세스 정보 (가상)
- **용도**: 커널과 사용자 공간 간 정보 전달
- **특징**: 실제 파일이 아닌 가상 파일시스템
- **포함**: 시스템 메모리, 장치, 하드웨어 설정 등
- **예시**: `/proc/cpuinfo` (CPU 정보)

### 📦 소프트웨어 및 마운트

#### `/opt` - 선택적 소프트웨어
- **용도**: 기본 설치에 포함되지 않은 대형 소프트웨어
- **특징**: 자체 포함된 애플리케이션
- **예시**: 서드파티 소프트웨어 패키지

#### `/mnt` - 임시 마운트 포인트
- **용도**: 외부 저장장치의 임시 마운트
- **예시**: 외부 하드드라이브, USB

#### `/media` - 이동식 미디어
- **용도**: USB 드라이브, CD-ROM 등 마운트
- **특징**: 데스크톱 환경에서 주로 사용
- **구조**: 각 장치별 하위 디렉토리

## 핵심 개념 및 보안

### 🔐 Root 계정

#### 기본 개념
- **Root 계정**: 리눅스 시스템에서 가장 강력한 사용자 계정
- **별칭**: superuser account라고도 불림
- **권한**: 시스템의 모든 명령어, 파일, 리소스에 대한 무제한 접근

#### 주요 특징
- 소프트웨어 설치/제거
- 시스템 전역 설정 변경
- 사용자 계정 관리
- 하드웨어 및 네트워크 설정

#### 보안 원칙
- **직접 로그인 금지**: root로 직접 로그인하지 않기
- **sudo 사용 권장**: `sudo` (superuser do) 명령어로 임시 권한 상승

#### 디렉터리 정보
- **Root 디렉터리**: `/` (최상위 디렉터리)
- **Root 홈 디렉터리**: `/root`

### 📂 파일 시스템 경로

#### 절대 경로 (Absolute Path)
- **정의**: 루트 디렉터리(/)부터 시작하는 완전한 경로
- **특징**: 현재 작업 디렉터리와 무관하게 파일 위치 명시
- **형식**: `/`로 시작
- **예시**: `/home/user/documents/report.txt`

#### 상대 경로 (Relative Path)
- **정의**: 현재 작업 디렉터리를 기준으로 하는 경로
- **특징**: 현재 위치에 따라 상대적으로 결정

##### 상대 경로 기호

| 기호 | 의미 | 설명 |
|------|------|------|
| `.` | 현재 디렉터리 | dot |
| `..` | 상위 디렉터리 | double dot |

##### 상대 경로 예시
- **같은 디렉터리 파일**: `document.txt`
- **하위 디렉터리**: `reports/report.txt`
- **상위 디렉터리**: `../fileInParentDir.txt`

### 💬 명령 프롬프트 (Command Prompt)

#### 정의
- **텍스트 기반 인터페이스**: 사용자가 운영체제와 상호작용하는 방식
- **기능**: 명령어 입력 → 텍스트 기반 피드백 수신
- **지원 OS**: Unix-like 시스템(Linux), Windows

#### 프롬프트 구성 요소
```bash
ubuntu@5e82576dec86:~$
```

| 구성 요소 | 설명 |
|-----------|------|
| `ubuntu` | 현재 사용자명 |
| `5e82576dec86` | 호스트명 (컴퓨터 이름) |
| `~` | 현재 디렉터리 (홈 디렉터리) |
| `$` | 프롬프트 기호 (일반 사용자) |

### 🔤 대소문자 구분 및 공백 사용 규칙

#### 대소문자 구분 (Case Sensitivity)
리눅스는 **대소문자를 구분**하며, 이는 다음 영역에 적용됨:
- 파일명
- 명령어
- 프로그래밍 언어

#### 공백 사용 제한
- **공백이 포함된 명령어**: 실행이 매우 어려움
- **권장사항**: 파일명과 디렉터리명에 공백 사용 지양

## 실무 활용 가이드

### ⭐ SWE가 알아야 할 핵심 포인트

#### 필수 디렉토리

1. **`/bin`, `/sbin`** 
   - 스크립팅 및 시스템 관리 작업

2. **`/etc`**
   - 서비스, 애플리케이션, 시스템 동작 설정
   - 설정 파일 수정 시 주요 작업 영역

3. **`/var`**
   - 로그 파일 분석 (`/var/log`)
   - 애플리케이션 모니터링 및 디버깅
   - 임시 파일 (`/var/tmp`)

4. **`/opt`**
   - 서드파티 소프트웨어 설치 위치

5. **`/home`**
   - 사용자 데이터 및 개인 설정 관리

### 🚨 주의사항

- **권한 관리**: 시스템 디렉토리는 root 권한 필요
- **로그 모니터링**: `/var/log` 정기적 확인 필수
- **디스크 용량**: `/var`, `/tmp`, `/home` 용량 관리 중요
- **백업**: 중요 설정은 `/etc` 백업 필수

### 💡 실무 활용 팁

#### 로그 분석
```bash
# 시스템 로그 확인
tail -f /var/log/syslog

# 애플리케이션 로그 위치
ls /var/log/
```

#### 설정 파일 백업
```bash
# 설정 변경 전 백업
sudo cp /etc/important-config /etc/important-config.backup
```

#### 시스템 정보 확인
```bash
# CPU 정보
cat /proc/cpuinfo

# 메모리 정보
cat /proc/meminfo

# 마운트된 파일시스템
cat /proc/mounts
```
