**핵심 내용**:

**주요 포인트**:
```yaml

- 메트릭 = 시스템 상태의 숫자 표현
- 쿠버네티스 환경의 특수성 (동적, 복잡)
- 공장 비유를 통한 직관적 이해
```

### 🔧 02편: 모니터링 파이프라인 완벽 이해
**파일**: `02_모니터링_파이프라인_완벽_이해.md`

**핵심 내용**:
- 파이프라인 개념 정복 (리눅스 파이프부터)
- 모니터링 파이프라인 3단계
- 각 구성요소 상세 분석
- 로그 파이프라인과의 비교

**주요 포인트**:
```yaml
- 파이프라인 = 순차적 작업의 연결
- 수집(Exporter) → 저장(Prometheus) → 시각화(Grafana)
- node-exporter: OS 통역사
- kube-state-metrics: 쿠버네티스 상태 번역기
```

### 🏆 03편: 프로메테우스 vs 다른 도구 비교

**핵심 내용**:

